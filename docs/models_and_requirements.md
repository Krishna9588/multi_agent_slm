# System Requirements & Model Selection Guide

This document provides a detailed breakdown of the models required to run the Autonomous Swarm, including alternatives, comparisons, and fallbacks.

## 1. Core System Requirements
Before installing models, ensure your local environment meets the following requirements:
- **Python 3.10+**: Required for async features and Playwright.
- **[Ollama](https://ollama.com/)**: The core engine for running local LLMs and SLMs.
- **[Docker Desktop](https://www.docker.com/)**: **CRITICAL.** Required by the `code_executor_agent`. Without Docker, the Swarm will not be able to safely execute Python code generated by the LLMs.
- **Playwright**: Installed via `pip install playwright && playwright install chromium` for browser automation.

---

## 2. Model Roles & Choices

Our architecture relies on three distinct "tiers" of models. To customize your swarm, you can pick one model from each tier based on your hardware capabilities.

### Tier 1: The Main Orchestrator (LLM)
**Role:** The "Brain". Responsible for understanding complex user queries, routing tasks using the `TransferToAgent` protocol, and summarizing final answers. It needs high reasoning capabilities and a large context window.

| Model | Size | Best For | Description & Link |
|---|---|---|---|
| **`llama3.1:8b`** (Default) | 4.7GB | Balanced Speed & Reasoning | Meta's state-of-the-art 8B model. Highly capable of strict JSON tool calling. [Docs](https://ollama.com/library/llama3.1) |
| **`gemma2:9b`** | 5.5GB | High Accuracy | Google's open-weights model. slightly heavier but excels at complex logical routing. [Docs](https://ollama.com/library/gemma2) |
| **`qwen2.5:7b`** | 4.0GB | Coding & Speed | Alibaba's model. Incredible at coding tasks and very fast. [Docs](https://ollama.com/library/qwen2.5) |

**Cloud Fallback:** If your local machine struggles with 8B models, you can run the system with the `--premium` flag to offload orchestration to Google's **Gemini 2.5 Flash** (requires `GEMINI_API_KEY` in `.env`).

---

### Tier 2: The Swarm Sub-Agents (SLM)
**Role:** The "Workers". These models power the `BrowserAgent`, `WebScraper`, and `CodeExecutor`. Because they are spawned dynamically and run repeatedly in loops, they must be extremely fast and lightweight (<3B parameters).

| Model | Size | Best For | Description & Link |
|---|---|---|---|
| **`llama3.2:3b`** (Default) | 2.0GB | General Sub-Tasks | Meta's new SLM. Fast enough to run in tight ReAct loops while clicking elements in the DOM. [Docs](https://ollama.com/library/llama3.2) |
| **`gemma4:e2b-mlx`** | ~2GB | Apple Silicon Macs | A specialized, highly optimized model for Mac unified memory. [Docs/Alternatives](https://ollama.com/library/gemma) |
| **`qwen2.5:1.5b`** | 930MB | Extreme Speed | Perfect for older laptops. Very fast, though it may occasionally hallucinate JSON syntax. [Docs](https://ollama.com/library/qwen2.5:1.5b) |

**Fallback Strategy:** The `core/swarm.py` router has a built-in memory module. If an SLM hallucinates a tool call or gets stuck in a loop due to low parameters, the Swarm Router injects a `[SYSTEM MEMORY WARNING]` to force it to recover.

---

### Tier 3: The Vision Agent (Multimodal)
**Role:** The "Eyes". Called by the `vision_agent.py` to analyze screenshots, solve visual CAPTCHAs, and interpret complex UI layouts.

| Model | Size | Best For | Description & Link |
|---|---|---|---|
| **`llama3.2-vision`** (Default) | 7.9GB | UI & CAPTCHAs | Meta's vision-enabled model. Excellent at extracting text from images and describing visual boundaries. [Docs](https://ollama.com/library/llama3.2-vision) |
| **`llava`** | 4.7GB | Lower VRAM | The open-source standard for vision. Slightly older architecture but runs well on 8GB RAM machines. [Docs](https://ollama.com/library/llava) |
| **`pixtral`** | 6.8GB | Document Parsing | Mistral's vision model. Highly tuned for reading documents and dense text in images. [Docs](https://ollama.com/library/pixtral) |

**Cloud Fallback:** If you cannot run a vision model locally due to VRAM constraints, the system will fall back to **Gemini 1.5 Flash** for image analysis (if `--premium` is enabled).

---

## 3. How to Configure Your Choices

Once you have pulled your chosen models (e.g., `ollama pull llama3.1:8b`), you must update the global configuration so the Swarm knows which models to use.

Open `.env` (create it if it doesn't exist) and set your preferred models:

```env
# Orchestrator Model
ORCHESTRATOR_MODEL=llama3.1:8b

# Swarm Sub-Agent Model (for Browser/Scraping)
SUB_AGENT_MODEL=llama3.2:3b

# Vision Model
VISION_MODEL=llama3.2-vision

# Cloud API Keys (for fallbacks and --premium mode)
GEMINI_API_KEY=your_google_ai_key
APIFY_TOKEN=your_apify_token
BRIGHTDATA_TOKEN=your_brightdata_token
```

If these environment variables are missing, the system will gracefully default to the models labeled "(Default)" in the tables above.
